# Greptile take home coding assessment

## Part 1: A developer-facing tool that allows devs to quickly AI-generate a changelog.

AI changelog generating cli tool - [clgen](https://github.com/Sanghawk/clgen)

Why I decided to go the CLI route:

- The tool is for developers. CLI allows them to generate these changelogs in the same workflow that they normally would.
- CLI is portable and easy set up. It works across multiple projects and can be easily installed through some python package manager.

How the clgen cli tool works:

- install directly onto your machine
- configure clgen cli settings like setting openai key and which model to choose
- generates changelogs from your local git project's commit history
  - you provide a commit or tag range
- writes out the changelog as an md file

Test it for yourself:

- Install the python package. Can use `venv` if you want to run it in isolated env.

```bash
python -m venv venv # creates virtual env
pip install "git+https://github.com/Sanghawk/clgen.git@main#egg=clgen" # install wheel
clgen --version # to test
```

- Configure openai api key and model.

```bash
clgen config openai set-key <openai-api-key>
clgen config openai get-key # check if it set correctly - only shows last 4 digits
clgen config openai set-model <openai-model-name> # default: gpt-4
clgen config openai get-model # check what default model you are using
```

Note: config file is locked by the user who created it. You can find `.config/clgen` folder created for this tool through:

```bash
# macOS / WSL / Linux
echo "XDG_CONFIG_HOME=$XDG_CONFIG_HOME"
```

- Before running `clgen generate` you should have a git initialized project to run it against.

```
/temp_folder
    /venv
    /some_git_project
```

```bash
# assuming you're in /temp_folder
clgen generate --repo-path ./some_git_project --since <commit hash | tag> --until <commit hash | tag> -o <output_file_name>.md
```

- To uninstall

```bash
pip uninstall -y clgen

# then find the .config/clgen folder and delete it
```

## Part 2: A simple public-facing website where the changelog appears for people to see.

First, run the development server:

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) with your browser to see the result (it will redirect to [http://localhost:3000/changelogs](http://localhost:3000/changelogs) by default).

`/changelogs` is the only web app path. It's a simple view of a changelogs generated by the first part of the assignment - [clgen](https://github.com/Sanghawk/clgen).
